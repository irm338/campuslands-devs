# Pipeline de Animación 3D

**Estudiante:** Irma Yaneht Arias Garcia

## Descripción
Este proyecto establece una estructura organizada para un flujo de trabajo de animación 3D, garantizando orden y escalabilidad.

## Estructura de directorios
- 01-modelado/ : Archivos editables de mallas y geometría.
- 02-texturas/ : Mapas, materiales y texturas.
- 03-renders/ : Salidas de imagen/video final.
- 04-entregas/ : Archivos finales para clientes o producción.
- docs/ : Documentación técnica.
- referencias/ : Material de apoyo y conceptos.

## Gestión de Versiones
Para evitar errores y distinguir pruebas de archivos finales:
- Se usará el sufijo `_v01`, `_v02` para versiones de trabajo.
- El sufijo `_FINAL` se reserva estrictamente para entregables aprobados.
- Todo material de prueba debe mantenerse en una subcarpeta `temp/` dentro de su respectiva categoría.
